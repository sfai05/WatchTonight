# WatchTonight — Design Context

## Design Context

### Users
Primarily the owner (self-hosted media enthusiast), with a public audience as a secondary goal. Context: sitting on the couch at home, deciding what to watch tonight. The job to be done is simple and specific — **scan recent releases, pick something good, start watching**. No deep research, no long reading sessions. Speed of decision matters.

### Brand Personality
**Nerdy · Cosy · Cinematic**

- *Nerdy*: Smart integrations (Radarr, Sonarr, Trakt), no hand-holding, assumes the user knows what they're doing
- *Cosy*: Warm amber tones, soft dark backgrounds — feels like settling in for a film, not browsing a database
- *Cinematic*: Editorial typography (Lora serif for titles), atmospheric poster backdrops, every detail treated like a movie credits sequence

Emotional goal: the faint excitement of *"ooh, what should I watch tonight?"*

### Aesthetic Direction
- **Theme**: Dark only — permanently. The deep navy (`#0d1321`) + amber (`#ffdca1`/`#ffb800`) palette is the identity. Light mode was removed as it doesn't fit the vibe.
- **Reference**: [movies.stevenlu.com](https://movies.stevenlu.com/) — minimal, poster-focused, gets out of the way
- **Anti-reference**: IMDb — too data-dense, too many ratings/lists/trivia. WatchTonight shows only what you need to decide.
- **Scope**: Only recent releases you can stream at home. No back-catalogue browsing, no deep filmography.
- **Typography**: Epilogue (display/headers), Lora (serif/dialog titles), Nunito (body). Intentional, not generic.

### Design Principles

1. **Decide fast, watch sooner** — Every design decision should reduce friction to picking a film. If a UI element doesn't help someone choose, it shouldn't be there.

2. **Posters first** — The poster wall is the product. Text and metadata support the poster; they never compete with it. Big images, atmospheric hover states.

3. **Warm but sharp** — The amber accent is warm and inviting, but the layout is precise and editorial. Never fussy, never clinical.

4. **Nerdy details, invisible UI** — Radarr/Sonarr links, Trakt, trailer embeds — power-user features that are present but unobtrusive. Reward curiosity, don't force it.

5. **Reasonable accessibility, no theatre** — WCAG AA contrast and keyboard navigation where it matters (interactive elements, dialogs). No need to gold-plate; it's a personal tool going public.

### Technical Constraints
- Vite + React 19, Tailwind CSS, shadcn/ui (Radix primitives), no animation libraries
- Dark-only CSS tokens in `:root` — no `.dark` class switching needed; `ThemeToggle` component should be removed
- Data is static JSON (TMDB + OMDb + Gemini summaries), updated on deploy
- Deployed to GitHub Pages at watchtonight.app
